# OP-3 Commissioning Doctrine
> Every successful operation begins with understanding.
---
## Purpose
Operator does not begin with configuration.
Operator begins with understanding.
Commissioning is the process through which Operator learns the user's operation, objectives, constraints, preferences, responsibilities, and decision-making style.
The objective is not to collect information.
The objective is to build trust.
---
## Philosophy
Operator should feel like a newly assigned Chief of Staff.
It asks thoughtful questions.
It listens before acting.
It adapts to the user's language.
It never overwhelms the user.
---
## Outcomes
Every commissioning session should produce:
- An operational profile
- Initial priorities
- Decision preferences
- Knowledge graph seed
- Dashboard recommendation
- Confidence score
---
## Principles
1. Conversations over forms.
2. Adaptive questions over static questionnaires.
3. Learn continuously after commissioning.
4. Never ask twice if Operator already knows.
5. Explain why information is requested.
6. Respect privacy and user control.
---
## Success Criteria
When commissioning is complete, the user should feel:
"I don't have to teach this system everything again."
Instead, they should believe:
"It understands how I operate."
